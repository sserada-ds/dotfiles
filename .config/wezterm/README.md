## WezTermとは

WezTermはRustで書かれた高性能なターミナルエミュレータです。

**主な特徴:**

| 特徴                 | 説明                                           |
| -------------------- | ---------------------------------------------- |
| GPUアクセラレーション | WebGPU/OpenGLによる高速レンダリング            |
| クロスプラットフォーム | macOS、Linux、Windows対応                      |
| Lua設定              | 柔軟で強力なLuaスクリプトによる設定            |
| マルチプレクサ内蔵   | tmuxのようなペイン分割・タブ管理機能           |
| リガチャ対応         | プログラミング用フォントのリガチャ表示         |
| Unicode完全対応      | 絵文字やNerd Fontsアイコンの完全サポート       |

## 基本操作

### ペイン操作

ペインは画面を分割して複数のターミナルを同時に表示する機能です。

| 操作             | キー            | 説明                           |
| ---------------- | --------------- | ------------------------------ |
| 水平分割         | `Cmd+d`         | 画面を左右に分割               |
| 垂直分割         | `Cmd+Shift+d`   | 画面を上下に分割               |
| 左へ移動         | `Cmd+h`         | 左のペインにフォーカス         |
| 下へ移動         | `Cmd+j`         | 下のペインにフォーカス         |
| 上へ移動         | `Cmd+k`         | 上のペインにフォーカス         |
| 右へ移動         | `Cmd+l`         | 右のペインにフォーカス         |
| ペインを閉じる   | `Cmd+w`         | 現在のペインを閉じる（確認あり）|
| ズーム           | `Cmd+z`         | 現在のペインを全画面表示/解除  |

### タブ操作

タブは複数のターミナルセッションを管理する機能です。

| 操作             | キー            | 説明                           |
| ---------------- | --------------- | ------------------------------ |
| 新しいタブ       | `Cmd+t`         | 新しいタブを開く               |
| 前のタブへ       | `Cmd+[`         | 左のタブに切り替え             |
| 次のタブへ       | `Cmd+]`         | 右のタブに切り替え             |
| タブ1へ          | `Cmd+1`         | 1番目のタブに切り替え          |
| タブ2へ          | `Cmd+2`         | 2番目のタブに切り替え          |
| ...              | `Cmd+3~8`       | 3~8番目のタブに切り替え        |
| 最後のタブへ     | `Cmd+9`         | 最後のタブに切り替え           |

### その他の操作

| 操作             | キー            | 説明                           |
| ---------------- | --------------- | ------------------------------ |
| 設定リロード     | `Cmd+Shift+r`   | 設定ファイルを再読み込み       |
| コピーモード     | `Cmd+Shift+x`   | Vimライクなテキスト選択モード  |
| コマンドパレット | `Cmd+Shift+p`   | 全コマンドの検索・実行         |
| デバッグ表示     | `Cmd+Shift+d`   | デバッグオーバーレイを表示     |

### マウス操作

| 操作             | 方法                  | 説明                           |
| ---------------- | --------------------- | ------------------------------ |
| URLを開く        | `Cmd+クリック`        | URLをデフォルトブラウザで開く  |
| テキスト選択     | ドラッグ              | テキストを選択してコピー       |
| ダブルクリック   | 2回クリック           | 単語を選択                     |
| トリプルクリック | 3回クリック           | 行全体を選択                   |

## 設定内容

### フォント設定

[設定ファイル](./appearance.lua)

| 設定項目       | 値                                           |
| -------------- | -------------------------------------------- |
| メインフォント | JetBrains Mono (Medium)                      |
| フォールバック | HackGen Console NF (日本語対応)              |
| フォントサイズ | 13.0pt                                       |
| 行間           | 1.1                                          |

**フォントフォールバックの仕組み:**

メインフォント（JetBrains Mono）で表示できない文字は、自動的に次のフォント（HackGen Console NF）で表示されます。これにより、英数字は美しいプログラミングフォントで、日本語は専用の日本語フォントで表示されます。

### カラースキーム

[設定ファイル](./appearance.lua)

| 設定項目         | 値                           |
| ---------------- | ---------------------------- |
| カラースキーム   | Gruvbox dark, hard (base16)  |

Neovimと統一したカラースキームを使用しています。

### ウィンドウ設定

[設定ファイル](./appearance.lua)

| 設定項目           | 値            | 説明                         |
| ------------------ | ------------- | ---------------------------- |
| 起動時サイズ       | ほぼ全画面    | 画面から20px余白を残す       |
| 背景透過度         | 92%           | 0%=完全透明、100%=不透明     |
| 背景ぼかし(macOS)  | 20            | 背景のぼかし強度             |
| ウィンドウ装飾     | RESIZE        | タイトルバーなし、リサイズ可 |
| パディング         | 8px           | 上下左右の余白               |

**起動時のウィンドウサイズ:**

起動時に画面いっぱいに表示されつつ、上下左右に20pxの余白が残り、壁紙が少し見える状態になります。余白のサイズは `appearance.lua` の `SCREEN_MARGIN` で調整可能です。

### タブバー設定

## 設定ファイル構造

```
.config/wezterm/
├── wezterm.lua      # メインエントリーポイント
├── appearance.lua   # 外観設定
├── keybinds.lua     # キーバインド設定
└── README.md        # このファイル
```

### 各ファイルの役割

| ファイル         | 説明                                           |
| ---------------- | ---------------------------------------------- |
| `wezterm.lua`    | エントリーポイント。モジュール読み込みと基本設定 |
| `appearance.lua` | フォント、カラー、ウィンドウ、タブ、ペインの設定 |
| `keybinds.lua`   | キーボードショートカットとマウス操作の設定     |

### モジュールパターン

各モジュールは `apply_to_config(config)` 関数をエクスポートします:

```lua
-- モジュールの例（appearance.lua）
local M = {}

function M.apply_to_config(config)
  config.font_size = 13.0
  -- 他の設定...
end

return M
```

```lua
-- メインファイル（wezterm.lua）での読み込み
require("appearance").apply_to_config(config)
require("keybinds").apply_to_config(config)
```

## カスタマイズ

### フォントを変更する

`appearance.lua` を編集:

```lua
config.font = wezterm.font_with_fallback({
  { family = "お好みのフォント", weight = "Medium" },
  { family = "日本語フォント", weight = "Regular" },
})
config.font_size = 14.0  -- サイズ変更
```

**利用可能なフォントを確認:**

```bash
wezterm ls-fonts
```

### カラースキームを変更する

`appearance.lua` を編集:

```lua
config.color_scheme = "Tokyo Night"  -- 例
```

**利用可能なカラースキームを確認:**

```bash
wezterm ls-fonts --list-colors
```

### 透過度を調整する

`appearance.lua` を編集:

```lua
config.window_background_opacity = 0.95  -- 0.0〜1.0
config.macos_window_background_blur = 30  -- macOSのみ
```

### 起動時のウィンドウサイズを調整する

`appearance.lua` 上部の `SCREEN_MARGIN` を編集:

```lua
local SCREEN_MARGIN = 20  -- 画面端からの余白（ピクセル）
```

- **余白を大きく**: 値を大きくする（例: 50）
- **余白を小さく**: 値を小さくする（例: 10）
- **完全に全画面**: 値を0にする

### キーバインドを追加する

`keybinds.lua` の `config.keys` テーブルに追加:

```lua
{
  key = "n",
  mods = "CMD|SHIFT",
  action = act.SpawnWindow,  -- 新しいウィンドウを開く
},
```

### 新しいモジュールを追加する

1. 新しいファイルを作成（例: `statusbar.lua`）
2. `apply_to_config` 関数を実装
3. `wezterm.lua` で読み込む

```lua
-- statusbar.lua
local M = {}
function M.apply_to_config(config)
  -- ステータスバーの設定
end
return M
```

```lua
-- wezterm.lua に追加
require("statusbar").apply_to_config(config)
```

## トラブルシューティング

### 設定が反映されない

1. 設定ファイルのパスを確認:
   ```bash
   ls -la ~/.config/wezterm/
   ```

2. 構文エラーをチェック:
   ```bash
   wezterm --config-file ~/.config/wezterm/wezterm.lua start
   ```

3. 設定をリロード: `Cmd+Shift+r`

### フォントが正しく表示されない

1. フォントがインストールされているか確認:
   ```bash
   wezterm ls-fonts
   ```

2. フォント名を正確に指定（大文字小文字を区別）

3. 必要なフォントをインストール:
   ```bash
   brew install --cask font-jetbrains-mono
   brew install --cask font-hackgen-nerd
   ```

### 日本語入力ができない

`wezterm.lua` で以下が設定されているか確認:

```lua
config.use_ime = true
```

### パフォーマンスが悪い

1. レンダリングバックエンドを変更:
   ```lua
   config.front_end = "OpenGL"  -- WebGpuの代わりに
   ```

2. FPSを下げる:
   ```lua
   config.max_fps = 60
   ```

### tmuxとの使い分け

WezTermはマルチプレクサ機能を内蔵しているため、基本的にtmuxは不要です。

| 機能         | WezTerm           | tmux              |
| ------------ | ----------------- | ----------------- |
| ペイン分割   | `Cmd+d`           | `prefix+%`        |
| タブ管理     | `Cmd+t`           | `prefix+c`        |
| セッション保持 | △ (限定的)      | ◎                 |
| リモートサーバー | ×              | ◎                 |

**使い分けの目安:**

- **ローカル作業**: WezTermのみで十分
- **リモートサーバー**: tmuxを使用（SSH切断時もセッション維持）
